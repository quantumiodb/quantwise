# 情景推演手册

本参考资料提供用于构建18个月情景的模板和最佳实践。
在情景分析时使用，以创建一致且高质量的情景。

## 情景构建的基本原则

### 1. MECE原则（Mutually Exclusive, Collectively Exhaustive）

情景应满足以下条件：
- **相互排斥**：各情景不重叠
- **完全穷尽**：覆盖所有主要可能性

### 2. 概率分配指南

| 情景 | 典型范围 | 分配依据 |
|---------|----------|-----------|
| Base Case | 50-65% | 最可能的发展 |
| Bull Case | 15-25% | 正面上行 |
| Bear Case | 20-30% | 负面下行 |
| 合计 | 100% | 必须调整至100% |

**非对称分配适用的情况：**
- Bull > Bear：利多因素较多的环境
- Bear > Bull：风险因素较多的环境
- Base > 60%：不确定性较低的情况
- Base < 50%：不确定性极高的情况（Base Case本身也不确定）

### 3. 时间线划分

**三阶段结构：**
- **0-6个月**：短期反应、初始行动
- **6-12个月**：中期发展、趋势形成
- **12-18个月**：长期结果、新均衡

## 按事件类型分类的情景推演手册

### 1. 货币政策事件（加息）

**Base Case（55%）：**
- 如预期实施加息
- 市场已基本消化
- 小幅股市下跌，债券收益率小幅上升

**Bull Case（20%）：**
- 加息幅度小于预期
- 鸽派前瞻指引
- 股市反弹

**Bear Case（25%）：**
- 加息幅度大于预期
- 鹰派前瞻指引
- 股市急跌，信用利差扩大

### 2. 地缘政治事件（冲突爆发）

**Base Case（50%）：**
- 冲突有限升级
- 短期大宗商品价格上涨
- 数月内局势稳定

**Bull Case（15%）：**
- 早期停火与和平协议
- 大宗商品价格正常化
- 市场早期恢复

**Bear Case（35%）：**
- 冲突长期化与扩大
- 大宗商品供应严重中断
- 全球通胀加速，经济衰退风险

### 3. 科技转型（AI监管）

**Base Case（50%）：**
- 渐进式监管引入
- 以行业自律为主
- 对创新的影响有限

**Bull Case（25%）：**
- 监管以有利于行业的形式制定
- 监管明确化反而加速投资
- 形成有利于大企业的进入壁垒

**Bear Case（25%）：**
- 引入严格监管
- 大幅限制AI开发
- 美国企业竞争力下降

### 4. 企业事件（大型并购）

**Base Case（60%）：**
- 获得监管机构批准
- 按计划完成交割
- 整合协同效应逐步实现

**Bull Case（15%）：**
- 协同效应超预期
- 整合顺利
- 后续并购战略奏效

**Bear Case（25%）：**
- 监管机构阻止或有条件批准
- 整合延迟或失败
- 协同效应未达预期

## 情景质量检查清单

### 内部一致性
- [ ] 各情景的假设条件是否逻辑一致
- [ ] 时间线发展的因果关系是否明确
- [ ] 经济指标预测是否相互一致

### 外部合理性
- [ ] 是否与过去的类似事件一致
- [ ] 是否恰当反映当前市场环境
- [ ] 是否与专家观点没有重大偏离

### 实用性
- [ ] 是否具有有助于投资决策的具体性
- [ ] 是否识别了可监控的催化剂
- [ ] 失效信号是否明确

### 完整性
- [ ] 是否包含主要风险情景
- [ ] 是否适当考虑了上行可能性
- [ ] 是否提及了尾部风险

## 常见错误及规避方法

### 1. 现状维持偏差
**问题**：给Base Case分配过高概率（70%以上）
**规避**：认识到从历史上看"什么都不变"的概率很低

### 2. 近因偏差
**问题**：过度评估最近事件的影响
**规避**：保持长期视角，参考过去的模式

### 3. 确认偏差
**问题**：仅采用与标题一致的解读
**规避**：刻意寻找相反意见

### 4. 过度精确
**问题**：对18个月后的数值预测到小数点
**规避**：承认不确定性，以范围表达

### 5. 情景重叠
**问题**：Base/Bull/Bear部分重叠
**规避**：明确各情景的边界条件

## 概率更新指南

收到新信息时的概率调整：

| 新信息性质 | 概率调整方向 |
|-------------|---------------|
| 支持情景的数据 | +5-15% |
| 与情景矛盾的数据 | -5-15% |
| 决定性证据 | +20-30% 或 -20-30% |
| 新风险因素出现 | Bear Case +5-10% |
| 风险因素消除 | Bear Case -5-10% |

**调整后必须重新调整使合计为100%**

## 输出质量标准

高质量情景的特征：
1. **具体性**：非抽象，包含数值、日期、名称
2. **逻辑性**：因果关系明确
3. **可验证性**：事后可判断对错
4. **实用性**：包含直接关联投资决策的信息
5. **谦逊性**：恰当表达不确定性
